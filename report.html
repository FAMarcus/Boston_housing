Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinao uma estatstica til no campo de anlise de regresso uma vez que descreve o quo "bom" a capacidade do modelo em fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo (ao utilizar o modelo) que pode ser explicada pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Questão 3 - Treinamento e Teste¶

  • Qual o benefcio de separar o conjunto de dados em alguma relao de subconjuntos de treinamento e de teste para um algoritmo de aprendizagem?

Dica: O que pode dar errado se no houver uma maneira de testar seu modelo?

Resposta:

Tendo em mos uma medida, queremos que todos os dados desta medida sejam bem representados pelo modelo escolhido. Um aspecto importante da medida que ela contenha dados suficientes que garantam que todos os pormenores do problema estejam bem especificados e que assim possam ser generalizados pelo modelo. Dividir os dados em uma grande frao para treino e uma frao menor para teste, permite estimar a performace do modelo em uma condio em que ele vai operar num momento em que novos dados forem inseridos, como feito com os dados no utilizados na fase de treinamento.

No entanto, a escolha destas fraes podem induzir ao erro. Se os dados estiverem ordenados, por exemplo, a frao dos dados para treino pode no ter informao alguma a respeito dos dados separados para teste, e isso pode levar o modelo a simplificar demais o problema, levando ao erro sistemtico. Uma maneira de se contornar o problema compartimentar os dados e ajustar os parmetros do modelo para diferentes combinaes destes compartimentos, garantindo o refinamento destes parmetros ajustados do modelo. Para isso utilizamos as tcnicas de cross-validation, utilizada na Questo 8. O concenso geral guardar 20% dos dados para validao do modelo e 80% para treinamento.

Subsequente validao, verifica-se se houve overfitting (alta varincia) ou underfitting (alto vis) no modelo, ajustando os parmetros que determinam a complexidade do modelo, minimizando o erro total.

Questão 8 - Validação Cruzada¶

  • O que a tcnica de treinamento de validao-cruzada k-fold?
  • Quais benefcios essa tcnica proporciona para busca em matriz ao otimizar um modelo?

Dica: Lembre-se de expllicar o que significa o 'k' da validao-cruzada k-fold, como a base de dados dividida e quantas vezes ela executada.

Assim como h um raciocnio por trs de utilizar um conjunto de teste, o que poderia dar errado ao utilizar busca em matriz sem um conjunto de validao cruzada? Voc pode utilizar a documentao para basear sua resposta.

Resposta:

  • A tcnica de validao-cruzada k-fold consiste em embaralhar os dados e distribu-los em 'k' grupos de 'n' elementos, usando um grupo para validao, outro para teste e o restante ('k-2') para treinamento. Treinado o modelo e obtido os melhores parmetros de ajuste, revezam-se os grupos para validao, teste e treinamento at que todas as combinaes sejam realizadas. Ao final faz-se uma anlise dos parmetros ajustados obtidos pelas diversas combinaes e escolhe um valor que melhor represente o problema resolvido.
  • Tem-se como benefcio evitar que um conjunto de validao e teste concentre acidentalmente uma peculiaridade especfica do conjunto de dados, gerando artificialmente parmetros que no represente o problema como um todo.

from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parmetro the 'max_depth' para uma 
        rvore de deciso de regresso treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validao-cruzada para o treinamento de dados
    cv_sets = ShuffleSplit(n_splits=10, test_size = 0.20, random_state=0)

    # TODO: Gerar uma rvore de deciso de regresso de objeto
    regressor = DecisionTreeRegressor(random_state=42) # 42 :-)

    # TODO: Gerar um dicionrio para o parmetro 'max_depth' com um alcance de 1 a 10
    params = {'max_depth':range(1,11)}

    # TODO: Transformar 'performance_metric' em uma funo de pontuao utilizando 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Gerar o objeto de busca em matriz
    grid = GridSearchCV(regressor, param_grid=params, scoring=scoring_fnc, cv=cv_sets)

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo timo
    grid = grid.fit(X, y)
    
    # Devolver o modelo timo depois de realizar o ajuste dos dados
    return grid.best_estimator_
